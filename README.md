# 斯特魯普實驗 (Stroop Test) - 專案說明

這是一個使用 **jsPsych 8.2.3** 建立的線上心理學實驗專案。此專案專為台灣繁體中文使用者設計，用於測量執行功能與注意力控制（即 Stroop Effect）。

## 實驗流程設計

目前的實驗包含以下兩個主要階段：

1. **第一階段：字體顏色反應**
   * **規則**：受試者需要忽略中文字義，只針對螢幕上出現的「字體顏色」做反應。
   * **刺激物**：單詞「紅、綠、藍、黃」以四種不同的顏色（紅、綠、藍、黃）呈現（共 16 種組合，每種組合重複 2 次，共 32 試次）。
   * **按鍵**：`1` (紅)、`2` (綠)、`3` (藍)、`4` (黃)。

2. **第二階段：文字字義反應**
   * **規則**：受試者需要忽略字體顏色，只針對螢幕上出現的「文字字義」做反應。
   * **刺激物**：與第一階段相同。
   * **按鍵**：`1` (紅)、`2` (綠)、`3` (藍)、`4` (黃)。

## 實驗資料與變數紀錄

實驗結束後，程式會自動將結果下載為名為 `stroop_data_[受試者編號].csv` 的 CSV 檔案。每個試次會記錄以下重要變數：

* `subject_id`: 受試者編號（在實驗開始時由受試者輸入）。
* `phase`: 實驗階段（`1` 代表第一階段，`2` 代表第二階段）。
* `word_meaning`: 刺激文字的字義（紅、綠、藍、黃）。
* `font_color`: 刺激文字的顏色（紅、綠、藍、黃）。
* `condition`: 情況（`一致 (congruent)` 或 `不一致 (incongruent)`）。
* `correct_response`: 系統預期的正確按鍵。
* `response`: 受試者實際按下的按鍵。
* `rt`: 反應時間（毫秒）。
* `correct`: 是否作答正確（`true` 或 `false`）。

## 技術細節

* **核心框架**: jsPsych v8.2.3
* **使用的插件 (Plugins)**:
  * `@jspsych/plugin-fullscreen`: 處理全螢幕模式的進入與退出。
  * `@jspsych/plugin-survey-text`: 用於收集受試者編號。
  * `@jspsych/plugin-html-keyboard-response`: 用於呈現刺激並收集鍵盤反應。
* **樣式 (CSS)**: 使用了微亮的灰色背景 (`#f5f5f5`) 以確保黃色字體能清晰顯示且不刺眼，並使用微軟正黑體確保中文顯示正常。

## 未來修改建議 (給 AI 或開發者)

若未來需要修改此專案，請參考以下方向：

1. **修改試次數量**: 若要增加或減少試次，可以修改 `phase1_procedure` 和 `phase2_procedure` 中的 `repetitions` 參數（目前設為 2）。
2. **修改呈現時間**: 若要更改刺激呈現的最長時間，可以修改 `trial_duration` 變數（目前為 3000ms）。
3. **增加練習階段 (Practice Block)**: 目前實驗直接進入正式測試。若需要，可以複製 `phase1_procedure` 建立一個練習組，並在其中加入作答回饋 (Feedback) 的邏輯。
4. **資料傳送**: 目前資料設計為本機下載 CSV 檔。若需將實驗部署至伺服器（例如 Pavlovia, JATOS 或自建後端），需要將 `jsPsych.data.get().localSave('csv', ...)` 的邏輯修改為 Ajax 或 fetch API。
